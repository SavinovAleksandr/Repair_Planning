# Сводный график ремонтов — автоматическая сборка

## Как работает

Скрипт `build_svod.py`:

1. Ищет в этой папке три файла:
   - `Проект Арх РДУ.xlsx` — выгрузка из ПК «Ремонты» для ОЗ Архангельского РДУ;
   - `Проект Коми РДУ.xlsx` — выгрузка из ПК «Ремонты» для ОЗ Коми РДУ;
   - `Приоритет строк по группам для сводного графика.xlsx` — справочник (можно оставить здесь навсегда).
2. Разбирает проекты, раскладывает строки по группам и сортирует их.
3. Сохраняет рядом файл `Сводный график ремонтов ЛЭП и сетевого оборудования на <месяц> <год> г.xlsx`.

Если какой-либо из проектов в корне отсутствует — скрипт попробует найти его в подпапке `Исходные материалы/` (удобно для отладки).

## Как пользоваться (Windows)

1. Положите в эту папку актуальные выгрузки `Проект Арх РДУ.xlsx` и `Проект Коми РДУ.xlsx`.
2. Дважды кликните `run_svod.bat` — откроется **окно с кнопками**.
3. Нажмите нужную кнопку — итоговый файл появится рядом с bat-ом.

### Окно (GUI)

Окно делится на логические блоки:

- **Сборка из проектов**
  - `Сформировать сводный (только объединение)` — чистое слияние Арх + Коми
    без сортировки по приоритетам, без оглавления, без Ганта, без фиксации
    высот. Используйте, чтобы быстро получить «черновик».
  - `Всё и сразу (пересобрать с нуля)` — полный конвейер: объединение →
    приоритеты → нормализация → оглавление → высоты → лист «Диаграмма».
  - Чекбокс `С нормализацией текста` — применяется к обеим кнопкам выше.
- **Применить к существующему своднику** — точечные правки поверх уже
  имеющегося в папке `Сводный график …xlsx`. Отмечайте чекбоксами, что нужно:
  - `Расстановка по приоритетам` — перечитывает сводник и перестраивает
    порядок строк (учитывает ручные правки дат и перемещения).
  - `Нормализация текста (H/N)` — только текстовые правки.
  - `Оглавление (TOC)` — перегенерировать строку-оглавление.
  - `Фиксация высот + wrap` — привести в порядок высоты заголовков и перенос
    строк в ячейках H/N.
  - `Диаграмма Ганта` — пересобрать лист «Диаграмма» по текущему Page1.
  - Кнопка `Выполнить отмеченное`.
- **Откат** — `Откатить к предыдущей версии` восстанавливает сводник из
  последней резервной копии в `backups/` (текущее состояние тоже уезжает в
  backups/ на всякий случай).
- **Лог** — подробный вывод внизу окна.

Перед любым сохранением предыдущая версия файла автоматически уезжает в
подпапку `backups/` с таймстампом в имени — «сломать» исторический результат
случайно не получится.

### Разовая подготовка машины с Windows

На машине должен быть установлен Python 3.10+ и пакет `openpyxl`. `tkinter`
входит в стандартную поставку Python для Windows — доустанавливать ничего не
нужно.

```bat
REM Если Python ещё не стоит, скачайте с https://www.python.org/downloads/
REM При установке — галочка "Add python.exe to PATH".

pip install openpyxl
```

После этого `run_svod.bat` работает двойным кликом — откроется окно.

### CLI-режим (для автоматизации / отладки)

`run_svod.bat` без аргументов открывает GUI. Если передать аргумент — запустится
консольный сборщик `build_svod.py`:

```bat
run_svod.bat --stage all
run_svod.bat --stage merge
run_svod.bat --stage sort
run_svod.bat --stage normalize
run_svod.bat --stage toc
run_svod.bat --stage heights
run_svod.bat --stage gantt
run_svod.bat --stage restore
run_svod.bat --dry-run
run_svod.bat --no-normalize --stage all
run_svod.bat --collapse-preamble --stage all
run_svod.bat --year 2026 --stage all
```

## Как пользоваться (Mac / Linux, для разработки)

```bash
cd "График ремонтов"
python3 -m pip install openpyxl              # однократно
python3 gui_svod.py                          # окно с кнопками

python3 build_svod.py                        # CLI: полная пересборка
python3 build_svod.py --stage merge          # только объединение
python3 build_svod.py --stage sort           # расстановка по приоритетам
python3 build_svod.py --stage normalize      # только нормализация
python3 build_svod.py --stage toc            # только оглавление
python3 build_svod.py --stage heights        # только высоты + wrap
python3 build_svod.py --stage gantt          # только лист «Диаграмма»
python3 build_svod.py --stage restore        # откат к последней копии
python3 build_svod.py --year 2026            # указать год явно
python3 build_svod.py --no-normalize         # без текстовой нормализации
python3 build_svod.py --collapse-preamble    # + сворачивать преамбулы
python3 build_svod.py --dry-run              # показать правки, ничего не сохранять
```

## Правила группировки и сортировки

Группы выводятся в следующем порядке (см. справочник приоритетов):

| № | Группа | Сортировка внутри |
|---|---|---|
| 1 | Ограничения ОЗ | сначала ОЗ Арх РДУ, потом Коми РДУ, внутри — по дате начала |
| 2 | ЛЭП 220 кВ | по дате начала |
| 3 | ПС 220 кВ | по порядку ПС из справочника, внутри одной ПС — по дате |
| 4 | ЛЭП 110 кВ | по дате начала |
| 5 | ПС 110 кВ | по порядку ПС из справочника, внутри ПС — по дате |
| 6 | Электростанции | по порядку из справочника, внутри — по дате |
| 7 | АЧР | по дате начала |

Правила классификации строки проекта:

- строка с названием «Ограничение режима работы …» → группа 1 (ОЗ определяется по источнику);
- строка, где поле наименования начинается с `ВЛ … 220 кВ …` → группа 2 (ЛЭП 220);
- строка, где поле наименования начинается с `ВЛ … 110 кВ …` → группа 4 (ЛЭП 110);
- строка с «Снижение объёма нагрузки, отключаемой действием АЧР …» → группа 7 (АЧР);
- иначе строка относится к подзаголовку, под которым она находится в проекте:
  - `ПС 220 кВ <имя>` → группа 3;
  - `ПС 110 кВ <имя>` → группа 5;
  - «… ТЭЦ» / «… ГРЭС» → группа 6;
  - остальное → группа «Прочее (не классифицировано)» в самом конце.

Если в колонке подгруппы встречается ПС / ТЭЦ, которой нет в справочнике приоритетов, — она пойдёт в конец своей группы, а в консоль будет выведено предупреждение. Это сигнал дописать справочник.

## Устройство выходного файла

Структура и оформление — такие же, как в проектах (шапка с грифом, подписи в конце, ширины колонок, объединения ячеек, стили шрифтов). Изменяется только:

- порядок строк (описан выше);
- заголовок в шапке — теперь «… операционной зоны Коми РДУ на <месяц> <год> г.»;
- добавляются строки-заголовки групп (`ЛЭП 220 кВ`, `ПС 220 кВ`, …) и, для групп 3/5/6, строки-подзаголовки объектов (`ПС 220 кВ Вельск`, `Сосногорская ТЭЦ`, …);
- сразу под шапкой появляется **строка-оглавление** с гиперссылками на первую строку каждой группы (`ЛЭП 220 кВ (стр. 12)` и т. п.);
- заголовки групп и подгрупп имеют фиксированную высоту (22 и 18 пт), у строк данных — автоподгонка по содержимому (с включённым wrap-text в H и N);
- в книге появляется второй лист **«Диаграмма»** — Гант-календарь:
  - строки = записи оборудования в том же порядке, что и на основном листе;
  - шкала по дням, охватывающая все заявки месяца (и любые «выходы» за границы месяца);
  - заливка ячеек — по виду ремонта (ТР / СР / КР / ВПр / ИСП / ЗРР / БВР), легенда справа;
  - выходные дни — светло-серым фоном;
  - первая строка и колонки «Объект / Вид» закреплены для прокрутки.

Содержимое самих строк (оборудование, даты, заявленные сроки) не меняется — только перегруппировывается. Поля «Причина/условия» (колонка H) и «Вид ремонта / примечание» (колонка N) дополнительно проходят текстовую нормализацию — см. ниже.

## Валидация исходных файлов

Перед разбором оба проектных файла проверяются на соответствие формату ПК «Ремонты» (лист `Page1`, шапка таблицы в строке 6: «Наименование оборудования», «Дата начала/окончания», «Вид ремонта»). Если формат не распознан — скрипт печатает понятное сообщение и завершается с кодом 3, не пытаясь разобрать файл.

## Текстовая нормализация полей H и N

Скрипт сам приводит к единому виду повторяющиеся ручные правки, которые эксперт делал раньше. По умолчанию применяются правила 1–4, правило 5 — опционально (`--collapse-preamble`).

| № | Правило | Действие |
|---|---|---|
| 1 | `с включением на ночь` / `без включения на ночь` | вырезается из H и дописывается к N (если ещё нет). |
| 2 | `с переводом на ОВ`, `с переводом на ОШВ`, `Совместно с …` | короткий хвост H переезжает в конец N. |
| 3 | `не в транзите`, `с отключением без разбоки разъединителями` | вырезается из H без переноса. |
| 4 | Общие подстановки | `ТДТ → точки деления транзита`, `А.Г.: ВЗ → А.Г.: ВЗ.`, `Включить → Включение`, `Вывести в ремонт → Вывод в ремонт`, `2026г → 2026 г.`, дефис-тире (`Ухта - Микунь` → `Ухта – Микунь`), пробелы в единицах (`110кВ → 110 кВ`, `2ч → 2 ч.`, `ч.30 м. → ч. 30 мин.`) |
| 5 | *(опционально)* `--collapse-preamble` | `Вывод в ремонт X для проведения <род.> Y` → `<Имен.> Y` (например, «Вывод в ремонт В-220 Т-1 для проведения испытаний В-220 Т-1» → «Проведение испытаний В-220 Т-1»). |

После сборки выводится краткий отчёт: сколько раз какое правило сработало. В режиме `--dry-run` дополнительно печатается детальный дифф по каждой изменённой строке (файл при этом не сохраняется — удобно проверить, что правило делает ожидаемое).

Отключить нормализацию целиком можно флагом `--no-normalize`.
